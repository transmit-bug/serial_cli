## 设计原则

### 核心原则

1. **渐进式复杂度**
   - 默认界面简洁，高级功能按需展开
   - 新用户快速上手，专家用户高效操作

2. **状态驱动**
   - 界面根据连接状态动态调整
   - 未连接/已连接/错误状态有明显视觉差异

3. **上下文完整性**
   - 相关功能在同一视图中完成
   - 减少跨界面操作的工作流

4. **容错性设计**
   - 危险操作有明确提示
   - 所有操作可撤销或确认

5. **性能优先**
   - 实时数据流不阻塞UI
   - 大数据量时自动降级显示

6. **桌面端优先**
   - 目标平台：macOS、Windows、Linux 桌面端
   - 不需要响应式设计，固定布局即可
   - 按钮大小、间距保持一致

## UI 组件库

### 技术栈

- **组件库**：shadcn/ui (New York style)
- **基础**：Radix UI + class-variance-authority
- **样式**：Tailwind CSS 4 + CSS 变量
- **图标**：Lucide React

### 已安装组件

```bash
frontend/src/components/ui/
├── badge.tsx       # 标签
├── button.tsx      # 按钮
├── card.tsx        # 卡片
├── dialog.tsx      # 对话框
├── input.tsx       # 输入框
├── label.tsx       # 标签
├── select.tsx      # 选择框
├── separator.tsx   # 分隔线
├── switch.tsx      # 开关
├── tabs.tsx        # 标签页
├── toggle.tsx      # 切换按钮
└── tooltip.tsx     # 提示框
```

### 使用规范

**导入方式**：
```tsx
import { Button } from "@/components/ui/button"
import { Input } from "@/components/ui/input"
import { Select, SelectContent, SelectItem, SelectTrigger, SelectValue } from "@/components/ui/select"
```

**按钮变体**：
```tsx
// 主要操作
<Button variant="default">连接</Button>

// 次要操作
<Button variant="secondary">取消</Button>

// 危险操作
<Button variant="destructive">断开</Button>

// 幽灵按钮
<Button variant="ghost">编辑</Button>

// 链接样式
<Button variant="link">查看详情</Button>
```

**按钮尺寸**：
```tsx
// 默认尺寸（用于主要操作）
<Button size="default">发送</Button>

// 小尺寸（用于工具栏）
<Button size="sm">清空</Button>

// 大尺寸（用于重要操作）
<Button size="lg">连接</Button>

// 图标按钮
<Button size="icon"><Trash2 className="h-4 w-4" /></Button>
```

### 组件规范

| 组件 | 推荐尺寸 | 使用场景 |
|------|----------|----------|
| Button | `size="sm"` 或 `size="default"` | 工具栏、表单 |
| Input | `className="h-8"` | 表单输入 |
| Select | `className="h-8"` | 下拉选择 |
| Badge | `variant="secondary"` | 状态标签 |
| Card | 默认 | 面板容器 |

#### RX数据显示区

**工具栏（精简到4个按钮）：**
```
┌─────────────────────────────────────────────────────────────────┐
│ [自动滚动☑] [HEX|▼]  [🔍搜索]  [清空] [⋮更多]                   │
└─────────────────────────────────────────────────────────────────┘
```

**按钮说明：**
- `[自动滚动☑]` - 切换开关，新数据自动滚动
- `[HEX|▼]` - 下拉选择显示格式（HEX/ASCII/混合/分栏）
- `[🔍搜索]` - 展开搜索栏
- `[清空]` - 清空当前数据
- `[⋮更多]` - 下拉菜单：导出、保存日志、显示设置

**搜索展开栏：**
```
┌─────────────────────────────────────────────────────────────────┐
│ 🔍 [搜索内容________] [上一个↓] [下一个↑] [区分大小写☐] [×关闭] │
│ 结果: 3/2341 匹配                                                │
└─────────────────────────────────────────────────────────────────┘
```

**数据显示格式：**

HEX模式：
```
[14:23:01] RX: 01 02 03 04 05 41 54 0D 0A
[14:23:02] TX: 41 54 2B 43 52 4C 46
```

ASCII模式：
```
[14:23:01] RX: ······AT..
[14:23:02] TX: AT+CRLF
```

混合模式：
```
[14:23:01] RX: 01 02 03 04 05  [· · · · ·]
[14:23:02] TX: 41 54 0D 0A      [A T · ·]
```

分栏模式：
```
[14:23:01] RX:
  HEX:     01 02 03 04 05 41 54 0D 0A
  ASCII:   ······AT..
```

## 项目特定优化

### 性能优化

**虚拟滚动：**
- 使用 `@tanstack/react-virtual`（已集成）
- RX数据区超过100项时启用
- 上下各预渲染20项

**内存管理：**
- RX数据最大10000行（FIFO清理）
- 脚本输出最大1000行
- 日志最大500行
- 达到80%内存时显示警告

**代码示例：**
```tsx
import { useVirtualizer } from '@tanstack/react-virtual'

function RxDataViewer({ packets }) {
  const parentRef = useRef<HTMLDivElement>(null)

  const virtualizer = useVirtualizer({
    count: packets.length,
    getScrollElement: () => parentRef.current,
    estimateSize: () => 40, // 每行约40px
    overscan: 20, // 预渲染上下各20项
  })

  return (
    <div ref={parentRef} className="h-[600px] overflow-auto">
      <div
        style={{
          height: `${virtualizer.getTotalSize()}px`,
          position: 'relative',
        }}
      >
        {virtualizer.getVirtualItems().map((item) => (
          <div
            key={item.key}
            style={{
              position: 'absolute',
              top: 0,
              left: 0,
              width: '100%',
              transform: `translateY(${item.start}px)`,
            }}
          >
            <DataPacketRow packet={packets[item.index]} />
          </div>
        ))}
      </div>
    </div>
  )
}
```

### 状态管理

**使用 Zustand：**
```tsx
import { create } from 'zustand'

interface ConnectionStore {
  status: 'disconnected' | 'connected' | 'error'
  port: string | null
  connect: (port: string) => Promise<void>
  disconnect: () => void
}

export const useConnection = create<ConnectionStore>((set) => ({
  status: 'disconnected',
  port: null,
  connect: async (port) => {
    set({ status: 'connected', port })

  },
  disconnect: () => {
    set({ status: 'disconnected', port: null })
  },
}))
```

### 错误处理

**统一错误处理：**
```tsx
import { toast } from 'sonner'

function handleError(error: Error, context: string) {
  console.error(`[${context}]`, error)

  if (error.message.includes('port already in use')) {
    toast.error('端口已被占用', {
      description: '请关闭其他使用该端口的程序',
      action: {
        label: '查看详情',

  } else {

    })
  }
}
```
