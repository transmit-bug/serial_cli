## 核心流程

### 流程1：快速连接串口并发送数据

**用户目标：** 快速连接串口设备并发送测试数据

```
┌─────────────────────────────────────────────────────────────┐
│ 1. 用户打开应用，进入"终端工作台"（默认界面）                │
└────────────────────┬────────────────────────────────────────┘
                     │
                     ▼
┌─────────────────────────────────────────────────────────────┐
│ 2. 界面显示"快速连接向导"（未连接状态）                      │
│    - 端口下拉: [COM1 ▼]                                      │
│    - 波特率: [9600 ▼]                                       │
│    - [高级配置 ▼]  (可选，展开显示完整参数)                  │
│    - [● 连接] 按钮（高亮，主要操作）                         │
│    - [⚙️ 端口管理] 按钮（次要操作）                          │
└────────────────────┬────────────────────────────────────────┘
                     │
                     ▼
┌─────────────────────────────────────────────────────────────┐
│ 3. 用户选择端口和波特率                                      │
│    - 点击[COM1 ▼]，从列表选择端口                            │
│    - 点击[9600 ▼]，从列表选择波特率                          │
│    - （可选）点击[高级配置 ▼]，调整数据位、校验位等          │
└────────────────────┬────────────────────────────────────────┘
                     │
                     ▼
┌─────────────────────────────────────────────────────────────┐
│ 4. 用户点击[● 连接]按钮                                      │
│    - 按钮显示加载状态："连接中..."                           │
│    - 界面显示Toast提示："正在连接 COM1..."                   │
└────────────────────┬────────────────────────────────────────┘
                     │
           ┌─────────┴─────────┐
           │                   │
           ▼                   ▼
┌─────────────────────┐  ┌─────────────────────┐
│ 5A. 连接成功         │  │ 5B. 连接失败         │
│     - 顶部状态栏变化: │  │     - 显示错误提示   │
│       "COM1 │ ●已连接"│  │     - Toast提示:     │
│     - 右侧辅助面板   │  │       "连接失败"      │
│       显示端口详情   │  │     - 端口下拉恢复    │
│     - 成功提示:      │  │       默认值          │
│       Toast: "✅ 已连接"│  │     - 错误面板显示   │
│       3秒后自动消失   │  │       详细信息和解决方案│
└─────────────────────┘  └─────────────────────┘
           │
           ▼
┌─────────────────────────────────────────────────────────────┐
│ 6. 界面切换到"已连接状态"                                    │
│    - 主工作区显示:                                           │
│      • RX数据显示区（顶部，60%）                             │
│      • TX发送区（底部，30%）                                 │
│    - 右侧辅助面板显示:                                       │
│      • 端口详情卡片                                          │
│      • 数据统计卡片                                          │
│      • 协议控制卡片                                          │
│    - 底部日志面板可折叠                                      │
└────────────────────┬────────────────────────────────────────┘
                     │
                     ▼
┌─────────────────────────────────────────────────────────────┐
│ 7. 用户在TX发送区输入数据                                    │
│    - 输入框显示: "输入要发送的数据..."                        │
│    - 点击[HEX|ASCII▼]切换格式                                │
│    - 点击[AT]快速插入"AT"                                    │
│    - 点击[CRLF]快速插入回车换行                              │
└────────────────────┬────────────────────────────────────────┘
                     │
                     ▼
┌─────────────────────────────────────────────────────────────┐
│ 8. 用户点击[发送]按钮                                        │
│    - 按钮显示加载状态:"发送中..."                            │
│    - RX区显示发送的数据（TX标识）                            │
│    - 右侧统计卡片更新: TX计数+1                              │
│    - （可选）输入框自动清空（如果勾选"发送后自动清空"）      │
└────────────────────┬────────────────────────────────────────┘
                     │
                     ▼
┌─────────────────────────────────────────────────────────────┐
│ 9. （可选）设备响应，数据显示在RX区                          │
│    - 新数据自动滚动到底部（如果勾选"自动滚动"）              │
│    - 每条数据显示: 时间戳、方向(RX/TX)、数据内容、字节数      │
│    - 右侧统计卡片更新: RX计数+1，流速更新                    │
└─────────────────────────────────────────────────────────────┘
```

**关键设计点：**
- ✅ 单一界面完成连接和发送，无需切换视图
- ✅ 清晰的状态转换（未连接→已连接→错误）
- ✅ 实时反馈（Toast提示、状态栏变化、统计更新）
- ✅ 快捷操作（[AT]、[CRLF]按钮）
- ✅ 自动化选项（自动滚动、自动清空）

### 流程2：配置协议处理器

**用户目标：** 为当前连接配置Modbus RTU协议

```
┌─────────────────────────────────────────────────────────────┐
│ 1. 前提条件: 用户已在"终端工作台"，串口已连接（COM1）        │
│    - 当前状态: 已连接                                        │
│    - 右侧辅助面板显示"协议控制卡片"                          │
└────────────────────┬────────────────────────────────────────┘
                     │
                     ▼
┌─────────────────────────────────────────────────────────────┐
│ 2. 用户在右侧"协议控制卡片"点击[选择协议▼]                  │
│    - 下拉菜单显示:                                          │
│      • 无                                                   │
│      • Modbus RTU                                           │
│      • AT Commands                                          │
│      • 自定义Lua协议                                        │
└────────────────────┬────────────────────────────────────────┘
                     │
                     ▼
┌─────────────────────────────────────────────────────────────┐
│ 3. 用户选择"Modbus RTU"                                     │
│    - 卡片右侧显示"[配置]"按钮                                │
│    - （可选）点击"快速配置常用协议"下的[配置]                │
└────────────────────┬────────────────────────────────────────┘
                     │
                     ▼
┌─────────────────────────────────────────────────────────────┐
│ 4. 打开协议配置抽屉（从右侧滑出，不是模态窗口）              │
│    ┌─────────────────────────────────────────────────────┐  │
│    │ ← 返回  Modbus RTU 配置                      [×关闭]│  │
│    ├─────────────────────────────────────────────────────┤  │
│    │ 从站ID: [1]                                         │  │
│    │ 范围:    [1] 到 [10]                                │  │
│    │                                                     │  │
│    │ ☑ 自动解析响应                                      │  │
│    │ ☑ 显示解析结果                                      │  │
│    │                                                     │  │
│    │ [启用]  [取消]                                      │  │
│    └─────────────────────────────────────────────────────┘  │
│    - 抽屉覆盖辅助面板的30%宽度                              │
│    - 背景略微暗化，但不是完全遮蔽                           │
└────────────────────┬────────────────────────────────────────┘
                     │
                     ▼
┌─────────────────────────────────────────────────────────────┐
│ 5. 用户配置参数                                             │
│    - 输入从站ID: 1                                          │
│    - 输入范围: 1-10                                         │
│    - 勾选"自动解析响应"                                     │
│    - 勾选"显示解析结果"                                     │
└────────────────────┬────────────────────────────────────────┘
                     │
                     ▼
┌─────────────────────────────────────────────────────────────┐
│ 6. 用户点击[启用]按钮                                        │
│    - 抽屉滑出关闭                                            │
│    - 协议控制卡片更新: "当前协议: Modbus RTU"                │
│    - Toast提示: "✅ Modbus RTU 协议已启用"                  │
│    - RX数据显示区增加解析结果列（如果勾选"显示解析结果"）    │
└────────────────────┬────────────────────────────────────────┘
                     │
                     ▼
┌─────────────────────────────────────────────────────────────┐
│ 7. 后续数据接收自动解析                                      │
│    - Modbus响应自动解析为寄存器值                            │
│    - RX区显示:                                              │
│      [14:23:01] RX: 01 03 0A 00 01 ...                      │
│      解析: Read Holding Registers, Addr:0x0000, Count:10   │
└─────────────────────────────────────────────────────────────┘
```

**关键设计点：**
- ✅ 协议配置整合在终端界面，不需要切换到"协议模板库"
- ✅ 使用抽屉而非模态窗口，保持上下文
- ✅ 配置后立即生效，实时反馈
- ✅ 可选的解析结果显示，不影响原始数据

### 流程3：循环发送数据

**用户目标：** 循环发送测试数据，观察设备响应

```
┌─────────────────────────────────────────────────────────────┐
│ 1. 前提条件: 用户已在"终端工作台"，串口已连接                │
└────────────────────┬────────────────────────────────────────┘
                     │
                     ▼
┌─────────────────────────────────────────────────────────────┐
│ 2. 用户在TX发送区输入数据                                   │
│    - 输入: "01 03 00 00 00 0A"（Modbus读取请求）            │
│    - 格式: HEX                                              │
└────────────────────┬────────────────────────────────────────┘
                     │
                     ▼
┌─────────────────────────────────────────────────────────────┐
│ 3. 用户点击[⚙️发送设置]按钮                                 │
│    - 展开发送工具抽屉（从底部滑出，不是侧边）                │

                     │
                     ▼

│ 4. 发送工具抽屉展开                                         │
│    ┌─────────────────────────────────────────────────────┐  │
│    │ ⚙️ 发送设置                                  [×关闭]│  │
│    ├─────────────────────────────────────────────────────┤  │

│    │                                                     │  │
│    │ 当前状态: ○未启动                                    │  │
│    │ [▶开始]  [⏸暂停]  [■停止]                           │  │
│    │                                                     │  │

                     │
                     ▼

└────────────────────┬────────────────────────────────────────┘
                     │
                     ▼
┌─────────────────────────────────────────────────────────────┐
│ 6. 用户配置循环参数                                         │
│    - 间隔: 100 ms                                           │
│    - 停止条件: 选择"手动"                                    │
│    - （可选）选择"N次后100次"或"时间后60秒"                  │
└────────────────────┬────────────────────────────────────────┘
                     │
                     ▼

│ 7. 用户点击[▶开始]按钮                                      │
│    - 整个顶部状态栏变化:                                    │
│      "COM1 │ ●已连接 │ [🔄循环发送中] [■紧急停止]"          │
│    - 整个TX发送区显示橙色脉冲边框                            │
│    - 循环发送卡片显示: "●正在运行 (已发送1次)"              │
│    - Toast提示: "⚠️ 循环发送已开始，按⌘.紧急停止"           │
│    - RX区开始显示接收的数据                                 │
│    - 右侧统计卡片实时更新                                   │
└────────────────────┬────────────────────────────────────────┘
                     │
                     ▼
┌─────────────────────────────────────────────────────────────┐
│ 8. 循环发送进行中                                           │
│    - 每100ms自动发送一次数据                                │
│    - 循环发送卡片计数器更新: "已发送234次"                   │
│    - 用户可以:                                              │
│      • 点击[⏸暂停]暂停发送                                 │
│      • 点击[■停止]停止发送                                  │
│      • 按快捷键⌘.紧急停止                                   │
│      • 调整间隔时间（实时生效）                              │
└────────────────────┬────────────────────────────────────────┘
                     │
           ┌─────────┴─────────┐
           │                   │
           ▼                   ▼
┌─────────────────────┐  ┌─────────────────────┐
│ 9A. 手动停止         │  │ 9B. 自动停止         │
│     - 点击[■停止]    │  │     - 达到设定条件    │
│     - 或按⌘.        │  │     (100次/60秒)     │
│     - 橙色边框消失   │  │     - 自动停止       │
│     - 状态栏恢复     │  │     - Toast提示:     │
│     - Toast: "已停止"│  │       "已完成100次"  │
└─────────────────────┘  └─────────────────────┘
```

**关键设计点：**
- ✅ 循环发送有强烈的视觉警告（橙色脉冲边框）
- ✅ 全局紧急停止（⌘.），在界面任何位置都可触发
- ✅ 实时计数器，让用户了解发送进度
- ✅ 多种停止条件（手动/次数/时间）
- ✅ 独立的发送工具抽屉，不干扰正常发送

### 流程5：编写和运行脚本

**用户目标：** 创建Lua脚本自动化串口操作

```
┌─────────────────────────────────────────────────────────────┐

                     │
                     ▼
```

## 流程优化建议

### 识别的痛点

1. **流程2（协议配置）**：
   - 痛点：从终端切换到"协议模板库"再返回
   - 解决：协议配置整合到终端界面的右侧辅助面板

2. **流程3（循环发送）**：
   - 痛点：循环发送时容易忘记停止，可能刷爆设备
   - 解决：强烈的视觉警告（橙色脉冲边框）、全局紧急停止（⌘.）

3. **流程5（脚本编辑器）**：
   - 痛点：输出区占用固定30%空间，大部分时间浪费
   - 解决：可调整分栏，默认折叠输出区

4. **流程7（跨界面操作）**：
   - 痛点：切换界面后看不到脚本运行状态
   - 解决：顶部状态栏显示后台任务指示器

### 流程性能指标

**目标：**
- 快速连接: 3次点击完成连接
- 协议配置: 不离开终端界面
- 脚本运行: 不阻塞终端操作
- 错误恢复: 1次点击重试连接
