## 🎯 设计原则

### 核心原则

1. **渐进式复杂度**
   - 默认界面简洁，高级功能按需展开
   - 新用户快速上手，专家用户高效操作

2. **状态驱动**
   - 界面根据连接状态动态调整（未连接/已连接/错误）
   - 状态转换清晰，视觉反馈及时

3. **上下文完整性**
   - 相关功能在同一视图中完成
   - 减少跨界面操作的工作流

4. **容错性设计**
   - 危险操作有明确提示
   - 所有操作可撤销或确认

5. **性能优先**
   - 实时数据流不阻塞UI
   - 大数据量时自动降级显示

## 🎨 核心界面概览

### 1. 终端工作台（主界面）

**设计目标：** 单一界面完成90%的串口操作

**关键特性：**
- 状态驱动设计（未连接/已连接/错误）
- RX数据显示区（60%）+ TX发送区（30%）
- 右侧辅助面板（30%）：端口详情、数据统计、协议控制、快捷操作
- 底部日志面板（可折叠）

**详细设计：** 参见 [UI-Design-Spec.md - 终端工作台](./UI-Design-Spec.md#1️⃣-终端工作台主界面)

---

### 2. 虚拟端口管理

**功能：** 创建和管理虚拟端口对，用于测试

**特点：**
- 实时流量图显示
- 实时数据预览
- 支持嗅探和回环测试

**详细设计：** 参见 [UI-Design-Spec.md - 虚拟端口](./UI-Design-Spec.md#2️⃣-虚拟端口管理)

---

### 3. 脚本编辑器

**功能：** Lua脚本编写、调试、运行

**特点：**
- 可调整分栏布局（拖动分隔线）
- Monaco编辑器（语法高亮、自动补全）
- 输出区默认折叠，运行时自动展开

**详细设计：** 参见 [UI-Design-Spec.md - 脚本编辑器](./UI-Design-Spec.md#3️⃣-脚本编辑器)

---

### 4. 协议模板库

**功能：** 管理预定义协议（Modbus、AT Commands等）

**特点：**
- 系统/用户协议分类
- 快速配置常用协议
- 支持导入/导出协议配置

**详细设计：** 参见 [UI-Design-Spec.md - 协议模板库](./UI-Design-Spec.md#4️⃣-协议模板库)

---

### 5. 设置

**功能：** 全局配置管理

**特点：**
- 标签页式布局（串口/显示/快捷键/通知/高级）
- 导入/导出配置
- 恢复默认设置

**详细设计：** 参见 [UI-Design-Spec.md - 设置](./UI-Design-Spec.md#5️⃣-设置界面)

## 🔄 关键用户流程

### 核心流程

1. **快速连接串口并发送数据**
   - 3次点击完成连接
   - 状态驱动的界面转换
   - 实时反馈和错误处理

2. **配置协议处理器**
   - 在终端界面直接配置
   - 不切换视图
   - 实时生效

3. **循环发送数据**
   - 独立工具抽屉
   - 强烈的视觉警告
   - 全局紧急停止（⌘.）

**详细流程：** 参见 [User-Flow-Diagrams.md](./User-Flow-Diagrams.md)

## 🧩 技术实现

### 推荐组件库

本UI设计应基于成熟的组件库实现，而非从零开始构建。

#### shadcn/ui + Radix UI（强烈推荐）

- **优势**：
  - 基于Radix UI的无障碍组件
  - 代码可复制到项目，完全可控
  - TypeScript原生支持
  - 与Tailwind CSS完美集成
- **官网**：https://ui.shadcn.com/

#### 其他选择

- **Chakra UI** - 快速开发，开箱即用
- **Mantine** - 企业级完整方案
- **Radix UI + Headless UI** - 高度定制化

### 技术栈

| 类别 | 推荐工具 |
|------|----------|
| **组件库** | shadcn/ui, Radix UI |
| **通知/Toast** | Sonner, react-hot-toast |
| **表单处理** | React Hook Form + Zod |
| **数据可视化** | Recharts, Visx |
| **编辑器** | Monaco Editor |
| **样式** | Tailwind CSS |

### 主题定制

**颜色系统（Cyber-Industrial主题）：**
```css
/* 信号色 - 主要操作 */
--signal-primary: #00ff41;

/* 警告色 */
--warning-primary: #ffb142;

/* 错误色 */
--error-primary: #ff4757;

/* 背景色 */
--bg-deepest: #0d0d0f;
--bg-base: #18181b;
```

**字体配置：**
```typescript
fonts: {
  mono: 'JetBrains Mono, SF Mono, monospace',  // 数据显示
  sans: 'system-ui, sans-serif',               // 界面文字
}
```

**实现示例：** 参见 [UI-Design-Spec.md - 技术实现指南](./UI-Design-Spec.md#技术实现指南)

## 🚀 性能优化

### 虚拟滚动

- RX数据区超过100项时使用虚拟滚动
- 动态计算行高，预渲染上下各20项
- 最大保存10000行数据（FIFO清理）

### 内存管理

- 脚本输出限制1000行
- 日志限制500行
- 达到80%内存时显示警告

**详细策略：** 参见 [UI-Design-Spec.md - 性能优化](./UI-Design-Spec.md#项目特定优化)

## ♿ 可访问性

本UI设计通过使用成熟组件库自动遵循 WCAG 标准。

### 自动支持（由组件库提供）

- **键盘导航** - Radix UI 自动处理
- **焦点管理** - 自动焦点陷阱和恢复
- **屏幕阅读器** - 自动 ARIA 属性
- **对比度** - Tailwind 无障碍调色板

### 全局快捷键

| 功能 | macOS | Windows/Linux |
|------|-------|--------------|
| 命令面板 | ⌘K | Ctrl+K |
| 快速打开 | ⌘P | Ctrl+P |
| 保存 | ⌘S | Ctrl+S |
| 运行脚本 | ⌘Enter | Ctrl+Enter |
| 紧急停止 | ⌘. | Ctrl+. |
